# Documentazione di design — AI-Dev Flow

Materiale di progettazione e presentazione del kit. La **fonte di verità operativa** resta il codice
del repo (manifest, skill, hook, `PROCESS.md`, `INSTALL.md`); questi documenti sono la vista d'insieme
e il razionale.

## Contenuto

- [`AI_Dev_Flow_Kit_MVP_Revisione.md`](AI_Dev_Flow_Kit_MVP_Revisione.md) — documento di revisione:
  architettura del kit come **plugin Claude Code abilitabile per-progetto**, con il *perché* di ogni
  scelta (skill, hook, install per-progetto, regola del 98%, architecture doc, test-playbook, snapshot gate).
- [`AI_Dev_Flow_Presentazione.md`](AI_Dev_Flow_Presentazione.md) — presentazione per la direzione:
  cosa è, cosa risolve, human-in-the-loop, controllo dei costi, piano in tre fasi.
- [`AI Dev Flow V4.drawio`](AI%20Dev%20Flow%20V4.drawio) — diagramma di processo (versione corrente):
  le 5 fasi, i 3 gate umani, i punti di automazione, la legenda dei meccanismi.
- [`AI Dev Flow.drawio`](AI%20Dev%20Flow.drawio) — diagramma di processo (versione precedente, storica).

> I `.drawio` si aprono con [draw.io](https://app.diagrams.net) o l'estensione Draw.io di VS Code.
